# openclaw-doc-textify — tasks

## 方針

- まずは **単票の新規書類取り込み** を最短で成立させる
- 次に **バッチ処理** を入れる
- その後に **既存画像マイグレーション** を入れる
- すべて Slack 前提で進める
- Notion 書き込みはレビュー承認後のみ行う

---

### T0-1. プロジェクト初期構成を決める
- [ ] 実装言語 / ランタイムを確定する
- [ ] ディレクトリ構成を決める
- [ ] `.env.example` を作る

### T0-2. 開発用の最低限セットアップ
- [ ] package manager / test runner / lint を決める
- [ ] 開発用スクリプトを定義する

- [ ] Slack 接続設定
- [ ] Notion 接続設定

## Phase 1 — 単票の新規書類取り込み

### T1-1. Slack 受信を実装する
- [ ] 書類画像を含む Slack メッセージを受信できるようにする
- [ ] 対象チャンネル / スレッドでの動作条件を決める
- [ ] file reference を取得できるようにする

### T1-2. 画像取得処理を実装する
- [ ] Slack ファイルをダウンロードできるようにする
- [ ] 画像の一時保存処理を作る
- [ ] 画像メタデータを保持する

### T1-3. OCR 呼び出しを実装する
- [ ] 単一画像に対する OCR 実行関数を作る
- [ ] OCR raw text を保存する
- [ ] OCR失敗時のエラー返却を定義する

### T1-4. 分類 / 構造化を実装する
- [ ] OCR結果から documentType / title / summary を生成する
- [ ] Notion 追加先候補を1件以上出せるようにする
- [ ] structuredContent を生成する
- [ ] confidence を返す

### T1-5. Notion 候補解決を実装する
- [ ] `📂 DOMAIN` の候補ページ一覧取得
- [ ] ページ title / description / domain を読み取る
- [ ] 候補ページを解決するロジックを作る

### T1-6. Slack レビュー表示を実装する
- [ ] 単票レビューのメッセージ形式を作る
- [ ] 少なくとも以下を表示する
  - [ ] 文書種別
  - [ ] 要約
  - [ ] 追加先
  - [ ] 追加予定内容
- [ ] OK / 修正 / キャンセル操作の導線を作る

### T1-7. レビュー状態管理を実装する
- [ ] `ReviewItem` を保存できるようにする
- [ ] `received` → `ocr_done` → `classified` → `review_pending` を追跡できるようにする
- [ ] 承認 / キャンセル後の状態更新を実装する

### T1-8. Notion 反映を実装する
- [ ] 承認済みアイテムを Notion に追加する
- [ ] 標準ブロック構造で書き込む
- [ ] 書き込み結果を状態に反映する

### T1-9. 完了メッセージを実装する
- [ ] Slack に反映完了メッセージを返す
- [ ] 失敗時は理由付きメッセージを返す

### T1-10. 単票 E2E テスト
- [ ] 1枚の画像を入力してレビューまで通す
- [ ] 承認後に Notion 反映されることを確認する
- [ ] キャンセル時に Notion 書き込みされないことを確認する

**完了条件**
- 1枚の書類画像について、Slack受信 → OCR → 分類 → レビュー → 承認 → Notion反映が通る

## Phase 2 — 修正フローと品質向上

### T2-1. 修正指示の再解釈を実装する

### T2-2. confidence 低値時の扱いを実装する

- [ ] ユーザーが手で選び直せるようにする

### T2-3. 失敗時の再試行導線を作る
- [ ] OCR失敗の再試行
- [ ] Notion書き込み失敗の再試行
- [ ] 再試行状態を保存する

**完了条件**

## Phase 3 — 複数画像バッチ処理

### T3-1. BatchSession を実装する

### T3-2. 複数画像受信を実装する
- [ ] 1メッセージ複数ファイル
- [ ] 連投メッセージの扱い方を決める
- [ ] 各画像を個別 ReviewItem に分割する

### T3-3. バッチサマリー表示を実装する
- [ ] 暫定タイトルと追加先候補一覧を表示する
- [ ] 次のレビュー開始導線を作る

### T3-4. 逐次レビューを実装する
- [ ] 1件ずつ詳細レビューを進める
- [ ] OK / 修正 / キャンセル / スキップを扱う
- [ ] 次アイテムへ進む制御を入れる

### T3-5. バッチ完了サマリーを実装する
- [ ] 登録件数
- [ ] キャンセル件数
- [ ] スキップ件数
- [ ] 失敗件数

### T3-6. バッチ E2E テスト
- [ ] 3〜5件程度でまとめて処理できることを確認する
- [ ] スキップが後続アイテムに影響しないことを確認する

**完了条件**
- 複数画像の一括入力に対して、Slack 上で逐次レビューしながら Notion 反映できる

### T4-4. マイグレーション用レビューを実装する

### T4-6. マイグレーション完了サマリーを実装する
