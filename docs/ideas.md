## 差分の観点（読み方ガイド）

| ツール | 特性 | 得意なこと |
|---|---|---|
| **Notion AI** | Notion ワークスペース内で完結。ドキュメント生成・要約・DB操作 | 社内ナレッジの検索・要約・ページ生成 |
| **Claude Code** | ターミナル常駐。コードベース全体を理解し編集・テスト・コミット | コード修正・リファクタ・PR作成・テスト |
| **Codex (OpenAI)** | クラウドサンドボックスで非同期コード実行。PR自動作成 | 並列バッチ的なコード修正・マイグレーション |
| **OpenClaw** | 24/7デーモン。メッセージングUI。メール/カレンダー/ブラウザ/IoT連携 | 生活・業務の自動化、常駐監視、スケジュール実行 |

**OpenClaw 固有の強み**:
- メッセージアプリ（Telegram/Slack等）から自然言語で指示できる
- Heartbeat による cron 的な定期実行（寝ている間に動く）
- メール・カレンダー・ブラウザ・ファイルシステム・IoT への直接アクセス
- 5,700+ のコミュニティスキル（ClawHub）
- モデル非依存（Claude, GPT, DeepSeek, Ollama 等を切替可能）
- サブエージェント生成による複雑タスクの分割実行

**概要**: 未読メールの要約・分類・返信ドラフト作成をエージェントに任せる

**具体的にやること**:

## 優先ユースケース（検討済み・2026-03-10）

議論を通じて、以下の3つのユースケースを優先候補として具体化した。

### 優先度の決定

| 優先度 | ユースケース | 理由 |
|---|---|---|
| **A（最優先）** | 書類→Notion 自動振り分け | 日常の最大のペインポイント。紙の書類が溜まる |
| **B** | 確定申告トラッキング | Aができれば自然と対応できる。住宅ローン初年度で今年が初の確定申告 |
| **C** | ふるさと納税 横断検索・管理 | 複数サイト横断が面倒。食品・日用品メイン、旅行もあり |

> A → B → C の順で実装する。AができればBはその延長線上にある。

### ユースケースA: 書類→Notion 自動振り分け（最優先）

**概要**: 紙の書類をスマホで撮影 → OCR + AI分類 → Notion の該当ページに自動追加 → Telegram で報告＆レビュー

**ペインポイント**:
- 幼稚園のお手紙、病院の書類、各種通知…紙の書類が日々溜まる
- 写真に撮っても「どこに保存するか」「何の書類だったか」が整理されない
- Notion に手動で振り分けるのが面倒で放置しがち

**処理フロー**:

```
📱 スマホで書類を撮影
    ↓
💬 Telegram に写真を送信
    ↓
🤖 OCR でテキスト抽出
    ↓
🧠 AI が内容を分析・分類
    ↓
📋 Notion の該当レコードのページに追加
    ↓
💬 Telegram で報告（何をどこにどう追加したかプレビュー表示）
    ↓
👤 ユーザーがレビュー
    ├── ✅ OK → 確定
    ├── ✏️ 修正指示 → 対話で修正
    └── ❌ キャンセル → 取り消し
```

**振り分け先**: Notion データベース「📂 DOMAIN」の全レコードが対象
- データベースURL: https://www.notion.so/04be990e6c5549598d237e71c2279c12?v=31dbef24c4bd80d7861f000c52fe0c2a
- data source ID: `collection://5d0a1f4f-ebb9-4fe7-af2f-b289c0472063`

**Notion DB 構造（実データから確認済み）**:

| プロパティ | 型 | 内容 |
|---|---|---|
| `Title` | title | レコード名（「狛江みずほ幼稚園」等） |
| `Domain` | select | Tax & Budget / Digital Assets / Life Base / Medical Records / Mothering / Career & Identity |
| `★` | select | ★★ / ★ / 〇 / archive（優先度・ステータス） |
| `description` | text | 説明文 |
| `Created` | created_time | 作成日時 |

**ページ内部の構造（実データから確認済み）**:
- 各レコードは Notion ページ。ページ内にブロックとして情報を追加している
- 現状の構造例（狛江みずほ幼稚園）:
  - `# 見出し` でセクション分け
  - `callout` ブロックでカテゴリ（例: 保育料引き落とし）
  - `<details>` トグルで折りたたみ
  - **画像（撮影した書類の写真）を直接ブロックに貼り付け**
- → この手動運用を自動化するのが本ユースケース

**書類の種類と運用**:
- お手紙（幼稚園からのお知らせが最多）
- 医療明細（通院履歴、処方記録）
- その他（保険、税金、住居関連の通知等）
- **まとめて撮る**: 溜まった複数の書類を一度に撮影して送る
  - → 複数枚の写真をバッチで受信し、**1枚ずつ別々の振り分け先に自動分類**する必要あり
  - → 各書類ごとに個別レビュー（OK/修正/キャンセル）

**Telegram での報告に求められる粒度**:
- 単なる「○○に追加しました」では不十分
- **ページのどこに、どんな内容を、どのように追加したか**をプレビュー表示する
- ユーザーがその場で内容を確認し、修正指示を出せるようにする
- 報告メッセージのイメージ（1枚ごと）:

```
📄 書類を認識しました（1/5）

種類: お知らせ（幼稚園）
内容: お餅つきについて（2025.12.2）
追加先: 狛江みずほ幼稚園
追加内容:
  - 12/17（水）お餅つき
  - 持ち物: お弁当 + タッパーウェア（3,4,5才）
  - 降園時間: 14時
  - 注意: 体調不良時は休む

[✅ OK] [✏️ 修正] [❌ キャンセル]
```

**バッチ処理時のフロー**:

```
📱 溜まった書類5枚をまとめて撮影 → Telegram に送信
    ↓
🤖 5枚それぞれをOCR + 分類
    ↓
💬 サマリー表示:
    「5枚の書類を認識しました:
     1. お餅つきについて → 狛江みずほ幼稚園
     2. 医療費明細 → 通院履歴
     3. 保険料控除証明 → 生命保険、医療保険
     4. ...
     5. ...」
    ↓
💬 1枚ずつ詳細プレビュー → レビュー
    ↓
👤 各書類ごとに OK / 修正 / キャンセル
```

**OpenClaw との関連**:
- ideas.md #9（経費レシート自動処理）の拡張版
- OCR + LLM のマルチモーダル処理
- Telegram → OpenClaw Gateway → Notion API の連携
- レビュー＆修正のインタラクティブな対話ループ

- [x] 撮影する書類の種類 → お手紙（幼稚園）、医療明細が中心
- [x] 1回に撮る枚数 → **溜まったドキュメントをまとめて撮る**（バッチ処理が必要）
- [x] 既存画像のテキスト化 → **画像ごとにレビューして判断**（一律変換しない）

**サブ機能: 既存画像のテキスト化マイグレーション**（2026-03-18 追加）

既存の Notion ページにすでにアップロードされているスキャン書類画像を、
OCR でテキスト化するマイグレーション機能。新規書類の振り分け（上記メインフロー）とは別に、
既存データのテキスト化を行う。

**背景**:
- 画像だと Notion の検索にヒットしない
- 画像が多いとページが重くなる
- Notion の画像表示が不安定になることがある
- ただし画像自体は残したい（確定申告の提出用など原本として価値がある）
- ビジュアライズ目的の画像（図表など）はテキスト化不要 → 画像ごとに判断が必要

**処理フロー**:

```
📋 Notion ページから画像ブロックを一括収集
    ↓
🔍 各画像に OCR を実行（テキスト抽出だけ先にやる）
    ↓
💬 Telegram にバッチ（5〜10件）でレビュー依頼を送信
    ↓
    ┌─────────────────────────────────┐
    │ 📄 ページ名: 狛江みずほ幼稚園    │
    │ [サムネイル画像]                  │
    │                                  │
    │ OCR結果:                          │
    │ 「12/17（水）お餅つき            │
    │   持ち物: お弁当 + タッパー ...」 │
    │                                  │
    │ [✅ テキスト化] [⏭ スキップ]     │
    └─────────────────────────────────┘
    ↓
✅ → テキストブロック追加 + 画像をトグル「原本画像」に移動
⏭ → 何もしない
```

**変換後の Notion ページ構造**:
- テキストブロックが追加され、検索可能になる
- 画像はトグル「原本画像」内に折りたたまれる（ページ軽量化）
- 画像は削除されない（提出用途に使える）

**詳細シナリオ**: `user-scenarios.md` のシナリオ 8-5 を参照
